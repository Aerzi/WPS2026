# 任务书：微软 Speaker Coach 竞品调研与演讲者模式可行性评估

## 1. 任务背景
作为 WPS 批注/评论功能的负责人，前期已梳理内部功能入口，设想构建“演讲者模式”以打通批注与人机协作。近期发现微软推出的 **Speaker Coach** 功能与此设想高度相关，需以此为切入点进行快速竞品分析与资料搜集。

## 2. 核心目标
1. **深度调研 Microsoft Speaker Coach**：利用 MCP 全面搜集其功能细节、交互逻辑、反馈维度及用户评价。
2. **资料沉淀**：将外部搜索到的竞品信息结构化保存至本地 `Research/` 目录。
3. **对比评估**：结合“演讲者模式+批注”的初始设想与微软现有方案，进行差异化分析。
4. **决策支撑**：基于初步调研结果，判断该方向是否值得投入资源进行更详细的深入调研。

## 3. 执行阶段规划

### 阶段一：竞品深度搜集与存档 (Competitor Analysis & Archiving)
**执行方式**: Web Search MCP + File Write
- **动作 1 (微软专项)**: 针对 Microsoft Speaker Coach (PowerPoint Presenter Coach) 进行全方位搜索：
    - **核心功能**: 实时/排练后反馈的维度（语速、填充词、敏感词等）。
    - **交互形态**: 在演示过程中如何提示？报告如何展示？
    - **技术原理**: 是否涉及 AI 语音识别、情感分析等。
- **动作 2 (行业拓展)**: 搜索“批注/评论”与“演讲模式”结合的其他竞品或学术案例，寻找“人机协作”在演讲场景下的应用。
- **动作 3 (存档)**: 将核心发现整理为 Markdown 文件（如 `Microsoft_Speaker_Coach_Analysis.md`）存入 `Research/`。

### 阶段二：设想与竞品碰撞 (Brainstorming & Gap Analysis)
**执行方式**: Context Reasoning
- **动作**: 对比微软 Speaker Coach 的实现路径与我们“演讲者模式连接批注”的设想。
- **分析维度**:
    - **重合度**: 微软是否已经覆盖了我们的核心设想？
    - **差异点**: 我们的“批注”切入点是否有独特的差异化价值（如多人协作批注辅助演讲）？
    - **机会点**: 微软方案中是否存在用户未满足的痛点？

### 阶段三：决策评估 (Decision Making)
**执行方式**: Evaluation
- **动作**: 综合竞品现状与自身优势，回答：
    - 是否需要对 Microsoft Speaker Coach 进行像素级拆解？
    - “演讲者模式”是否具备足够的产品竞争力？
- **输出**: 
    - **无需深入**: 竞品已垄断或方向不可行。
    - **需详细调研**: 发现差异化机会，需进一步制定详细调研计划（如用户访谈、技术可行性验证）。
