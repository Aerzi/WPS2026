数据仓库

ODS

Operational Data Store
  • 贴源层 / 原始数据
    保持与业务系统数据结构一致，不做修改。
  • 数据备份
    起到数据缓冲区和隔离区的作用。
  • 全量/增量
    存储历史数据，支持数据追溯。
Input: 业务DB / 日志

DWD

Data Warehouse Detail
  • 明细数据层
    进行数据清洗、脱敏、规范化处理。
  • 维度建模
    采用星型模型，构建事实表与维度表。
  • 统一标准
    统一字段命名、字典定义和编码格式。
Process: 清洗 / 建模

DWS

Data Warehouse Service
  • 服务数据层
    基于业务主题进行轻度汇总和聚合。
  • 宽表构建
    将多张关联表合并为主题宽表，减少Join。
  • 公共指标
    沉淀通用的业务指标，避免重复计算。
Process: 聚合 / 汇总

ADS

Application Data Store
  • 应用数据层
    面向具体业务应用的最终结果数据。
  • 高度聚合
    数据量小，查询响应速度极快。
  • 多场景支持
    支持报表、仪表盘、用户画像等应用。
Output: BI / 报表
